# 校园食堂智能服务系统（商务智能课程大作业）

## 1. 项目简介

本项目是一个面向高校食堂场景的 **校园食堂智能服务系统**，基于商务智能（Business Intelligence）思路，围绕“学生端服务 + 食堂管理端运营 + 后勤决策端治理”三类用户建设完整的数据分析与决策支持闭环。

与传统只做图表展示的食堂分析项目相比，本系统强调：

- 从单角色分析升级为多角色可用系统；
- 从“看数据”升级为“给建议、能行动”；
- 从“描述历史”扩展到“预测未来 + 规范决策”。

本系统覆盖了课程中强调的三类分析能力：

1. 描述性分析：经营看板、销售分析、高峰分析、多食堂对比、个人消费报告；
2. 预测性分析：菜品销量预测、拥挤度估计；
3. 规范性分析：智能推荐、预算健康建议、促销建议、备餐建议、自动报告。

---

## 2. 项目背景

高校食堂每天会产生大量交易数据，例如：

- 学生匿名编号；
- 消费时间；
- 食堂与窗口；
- 菜品、价格、数量、金额；
- 支付方式、评分。

这些数据若仅用于收银存档，价值有限。  
若通过商务智能方法挖掘其中规律，可回答以下关键问题：

- 哪些菜品最受欢迎？哪些逐步滞销？
- 午餐/晚餐高峰何时出现？如何提前错峰备餐？
- 不同学生群体有哪些消费差异？
- 未来几天某菜品大约需要准备多少份？
- 如何给学生推荐更适合其预算与偏好的菜品？
- 后勤部门如何对多食堂进行统筹优化？

因此，本项目目标是建设一个“可运行、可分析、可推荐、可决策”的系统，实现从经验管理向数据驱动管理转型。

## 3. 项目目标

### 3.1 总体目标

构建一个覆盖学生端、管理端、后勤端的校园食堂智能服务系统，实现：

## 6. 系统功能模块详解（按 12 页）

## 6.1 学生端页面

### 页面1：`1_学生端_今日吃什么.py`

目标：为学生提供可解释的实时选餐建议。

输入：

- 学生编号；
- 预算（10元以内/10-15元/15-20元/不限）；
- 时段（早餐/午餐/晚餐/夜宵）；
- 口味（不限/清淡/辣/甜/咸）；
- 目标（吃饱/减脂/高蛋白/省钱/尝鲜）；
- 食堂偏好。

输出：

- TopN 推荐菜品；
- 每个菜品的价格、食堂、窗口、评分；
- 推荐理由（偏好匹配+评分+热度+时段适配）。

增强功能：

- 拥挤度估计；
- 避峰建议（排队时间等级）。

### 页面2：`2_学生端_我的消费报告.py`

目标：让学生了解自身消费结构和行为特征。

输出：

- 累计消费、消费单数、平均客单价；
- 常去食堂、常吃菜品；
- 每日消费趋势；
- 时段偏好图；
- 用户标签（如早餐党/夜宵党/省钱达人）。

### 页面3：`3_学生端_预算与健康建议.py`

目标：帮助学生进行预算管理与饮食优化。

功能：

- 月预算输入与实时跟踪；
- 月底消费外推预测；
- 超预算预警；
- 最近 7-30 天饮食结构分析；
- 健康建议（清淡/高糖/油炸等倾向提示）。

### 页面4：`4_学生端_菜品评分反馈.py`

目标：构建学生反馈闭环。

功能：

- 收藏/不喜欢菜品偏好；
- 菜品多维评分（口味/分量/价格/服务）；
- 文本评论；
- 上新投票与结果展示。

## 6.2 管理端页面

### 页面5：`5_管理端_经营数据看板.py`

- 价格-销量散点关系。

### 页面7：`7_管理端_消费高峰分析.py`

目标：做高峰管理和排班备餐支持。

功能：

- 时段订单与销售分析；
- 30 分钟粒度订单趋势；
- 星期-小时热力图；
- 高峰时段提示。

### 页面8：`8_管理端_关联规则分析.py`

目标：发现菜品搭配规律，用于套餐与联动销售。

功能：

- 支持度/置信度/提升度阈值调整；
- 规则表展示；
- 规则分布可视化；
- 指定菜品搭配推荐。

目标：预测未来销量，支持采购与备餐计划。

功能：

- 模型指标展示（MAE、RMSE、R²）；

- 单菜品趋势曲线。

### 页面10：`10_管理端_备餐建议.py`

目标：将预测结果转为可执行计划。

## 8. 核心算法与技术路线

## 8.1 技术路线总览

```text
数据生成/导入
   ↓
数据清洗与质量检测
   ↓
描述性分析（销售/时段/对比）
   ↓
智能推荐 + 关联规则
   ↓
销量预测
   ↓
备餐建议
   ↓
自动报告输出
```

## 8.2 智能推荐评分思路（学生端）

推荐分由多因子加权构成：

1. 评分因子；
2. 热度因子；
3. 时段适配；
4. 目标匹配；
5. 历史偏好匹配；
6. 不喜欢菜品惩罚（过滤）。

## 8.3 拥挤度估计逻辑

按 `食堂 + 工作日 + 15分钟时隙` 统计历史订单均值与波动，估计当前时段压力，映射拥挤等级：

- 低：明显低于均值；
- 中：接近均值；
- 高：高于均值约 30%；
- 爆满：高于均值约 60%。

## 8.4 关联规则（Apriori）

步骤：

1. 订单按 `order_id` 聚合成购物篮；
2. One-hot 编码；
3. 频繁项集挖掘；
4. 生成规则并按支持度、置信度、提升度筛选；
5. 输出可解释搭配建议。

## 8.5 销量预测（随机森林）

特征包含：

- `dish_name`、`category`、`canteen`、`window`；
- `weekday`、`is_weekend`；
- 时序特征：`lag_1`、`lag_7`、`rolling_mean_7`。

评估指标：

- MAE；
- RMSE；
- R²。

## 8.6 备餐建议规则

基本规则：

1. 建议备餐量 = 预测销量 × 安全系数（系统内置）；
2. 按预测区间划分风险；
3. 输出“增加备餐/保持/减少并促销”等动作建议。

## 8.7 自动报告生成

将核心指标、热门菜品、高峰时段、预测与建议汇总为 Markdown 报告，支持存档与二次编辑。
