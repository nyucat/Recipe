# 校园食堂消费分析与菜品推荐系统

## 1. 项目简介

本项目是一个面向校园食堂场景的商务智能分析系统，旨在通过对学生食堂消费数据进行采集、清洗、统计分析、关联规则挖掘、用户聚类、菜品推荐和销量预测，帮助食堂管理者了解学生消费行为、优化菜品供应、减少食物浪费，并提升学生就餐体验。

系统主要围绕校园食堂消费数据展开分析，支持数据导入、经营看板、菜品销售分析、消费时段分析、菜品关联分析、学生消费画像、菜品推荐、销量预测和备餐建议等功能。

本项目适合作为《商务智能》课程结课大作业，体现了描述性分析、预测性分析和规范性分析三类商务智能分析思想。

---

## 2. 项目背景

高校食堂每天会产生大量消费数据，包括学生消费时间、菜品名称、菜品价格、订单金额、消费窗口、食堂名称等。这些数据如果只作为交易记录保存，无法发挥更大的管理价值。

通过商务智能技术，可以从这些消费数据中发现学生的就餐偏好、菜品销售规律、窗口经营状况和未来销量趋势。例如：

- 哪些菜品最受学生欢迎？
- 哪些菜品经常被一起购买？
- 哪些时间段食堂最拥挤？
- 不同学生群体有哪些消费差异？
- 明天每种菜大概需要准备多少份？
- 如何为学生推荐更符合其偏好的菜品？

因此，本项目希望设计并实现一个校园食堂消费分析与菜品推荐系统，帮助食堂从经验管理转向数据驱动管理。

---

## 3. 项目目标

本项目主要目标如下：

1. 对校园食堂消费数据进行清洗、整理和结构化存储。
2. 构建食堂经营数据看板，展示核心经营指标。
3. 分析菜品销量、销售额、消费时段和窗口经营情况。
4. 使用关联规则算法挖掘菜品之间的搭配关系。
5. 使用聚类算法构建学生消费画像。
6. 根据热门程度、关联规则和用户画像实现菜品推荐。
7. 使用机器学习模型预测未来菜品销量。
8. 根据预测结果生成备餐和经营优化建议。
9. 通过可视化网页系统展示分析结果。

## 4. 系统用户

### 4.1 食堂管理人员

食堂管理人员可以查看经营看板、菜品销量、消费高峰、预测结果和备餐建议，用于优化食堂运营。

### 4.2 食堂窗口经营者

窗口经营者可以查看本窗口菜品销量、热门菜品、滞销菜品和推荐备餐量，从而调整生产计划。

### 4.3 学生用户

学生可以输入自己的匿名用户编号，查看个性化菜品推荐结果。

### 4.4 学校后勤管理部门

后勤部门可以通过系统了解不同食堂的整体运营情况，为食堂资源配置、窗口规划和服务优化提供依据。

## 5.5 菜品关联分析模块

该模块使用 Apriori 或 FP-Growth 算法，挖掘菜品之间的关联购买关系。

### 基本思想

每一笔订单可以看作一个购物篮，订单中的菜品可以看作购物篮中的商品。通过分析大量订单，可以发现学生经常一起购买的菜品组合。

### 示例

订单数据：

| 订单编号 | 菜品 |
|---|---|
| O001 | 米饭、红烧肉、紫菜汤 |
| O002 | 包子、豆浆、鸡蛋 |
| O003 | 麻辣香锅、米饭、饮料 |

系统可能挖掘出以下规则：

- 购买“包子”的学生经常同时购买“豆浆”。
- 购买“米饭”的学生经常同时购买“红烧肉”。
- 购买“麻辣香锅”的学生经常同时购买“饮料”。

### 输出指标

| 指标 | 含义 |
|---|---|
| support | 支持度，表示该组合在全部订单中出现的频率 |
| confidence | 置信度，表示购买前项菜品后也购买后项菜品的概率 |
| lift | 提升度，表示两个菜品之间关联强度 |

### 功能说明

- 支持设置最小支持度。
- 支持设置最小置信度。
- 支持设置最小提升度。
- 支持生成频繁项集。
- 支持生成关联规则。
- 支持展示菜品组合推荐。
- 支持根据单个菜品推荐搭配菜品。

### 应用价值

该模块可以用于设计套餐、组合促销、窗口联动销售和菜品推荐。

## 5.6 学生消费画像模块

该模块使用 K-means 聚类算法，根据学生消费行为将学生划分为不同群体。

### 用户特征

系统为每个学生构建以下特征：

- 总消费金额
- 消费次数
- 平均客单价
- 最近一次消费距今天数
- 早餐消费比例
- 午餐消费比例
- 晚餐消费比例
- 夜宵消费比例
- 偏好菜品类别
- 高价菜消费比例
- 低价菜消费比例

### 聚类流程

1. 按学生编号聚合消费数据。
2. 构造用户消费特征。
3. 对数值特征进行标准化。
4. 使用 K-means 进行聚类。
5. 分析每个群体的消费特征。
6. 为不同群体命名。
7. 生成不同群体的推荐策略。

### 用户群体示例

| 群体名称 | 特征描述 | 推荐策略 |
|---|---|---|
| 高频刚需型 | 消费次数多，客单价中等 | 推荐日常套餐 |
| 低频高消费型 | 消费次数少，单次消费金额高 | 推荐精品套餐 |
| 早餐依赖型 | 早餐消费占比高 | 推荐早餐组合 |
| 夜宵偏好型 | 夜间消费明显 | 推荐夜宵菜品 |
| 价格敏感型 | 偏好低价菜品 | 推荐优惠套餐 |
| 健康饮食型 | 偏好素菜、汤类、轻食 | 推荐健康菜品 |

## 5.7 菜品推荐模块

该模块用于向学生推荐菜品。

### 推荐方式一：热门菜品推荐

适用于没有历史消费记录的新用户。

推荐依据：

- 菜品销量
- 菜品评分
- 菜品复购率
- 菜品销售增长率

### 推荐方式二：关联规则推荐

适用于已经选择某个菜品的用户。

示例：

- 用户选择“包子”，系统推荐“豆浆”。
- 用户选择“米饭”，系统推荐“红烧肉”和“紫菜汤”。
- 用户选择“麻辣香锅”，系统推荐“饮料”。

### 推荐方式三：用户画像推荐

适用于有历史消费数据的学生。

示例：

- 早餐型用户推荐包子、豆浆、粥类。
- 夜宵型用户推荐炒饭、面条、饮品。
- 价格敏感型用户推荐低价套餐。
- 健康型用户推荐素菜、轻食、汤类。

### 推荐结果展示

系统输出：

- 推荐菜品名称
- 所属窗口
- 菜品价格
- 历史评分
- 推荐理由
- 推荐分数

## 5.8 菜品销量预测模块

该模块使用机器学习模型预测未来菜品销量。

### 预测目标

- 预测明日某菜品销量。
- 预测未来 7 天某菜品销量。
- 预测某窗口未来订单量。
- 预测某食堂未来销售额。

### 特征设计

| 特征 | 含义 |
|---|---|
| dish_name | 菜品名称 |
| category | 菜品类别 |
| price | 菜品价格 |
| weekday | 星期几 |
| is_weekend | 是否周末 |
| lag_1 | 前一天销量 |
| lag_7 | 前 7 天销量 |
| rolling_mean_7 | 过去 7 天平均销量 |
| canteen | 食堂名称 |
| window | 窗口名称 |

### 可选模型

- 线性回归
- 随机森林回归
- XGBoost 回归
- LSTM 时间序列模型

本项目默认使用随机森林回归模型，原因是随机森林能够处理非线性关系，对异常值不太敏感，且实现难度适中。

### 模型评价指标

- MAE
- RMSE
- R²

### 输出结果

- 预测销量表
- 真实销量与预测销量对比图
- 未来 7 天销量趋势图
- 模型评价指标
- 预测误差分析

### 核心算法说明
#### 关联规则算法

本项目使用 Apriori 或 FP-Growth 算法挖掘菜品之间的关联关系。

主要步骤：

将每个订单转换为菜品集合。
对菜品进行 One-Hot 编码。
设置最小支持度。
挖掘频繁项集。
根据频繁项集生成关联规则。
根据置信度和提升度筛选规则。
根据规则生成菜品搭配推荐。

应用场景：

套餐设计
搭配推荐
组合促销
窗口联动销售
#### K-means 聚类算法

本项目使用 K-means 算法对学生消费行为进行分群。

主要步骤：

按学生编号聚合订单数据。
构建用户消费特征。
对特征进行标准化。
使用 K-means 聚类。
分析每一类用户特征。
给每一类用户命名。
根据用户群体生成推荐策略。

应用场景：

学生消费画像
个性化推荐
精准营销
食堂服务优化
#### 销量预测模型

本项目使用随机森林回归模型预测菜品销量。

主要步骤：

按日期和菜品统计历史销量。
构造时间特征和滞后特征。
划分训练集和测试集。
训练随机森林回归模型。
预测未来销量。
使用 MAE、RMSE、R² 评价模型效果。
根据预测结果生成备餐建议。

应用场景：

备餐预测
库存管理
缺货预警
减少浪费

## 项目创新点
将商务智能技术应用到校园食堂管理场景，具有较强现实意义。
同时结合描述性分析、预测性分析和规范性分析。
使用关联规则算法挖掘菜品搭配关系，可用于套餐设计。
使用聚类算法构建学生消费画像，实现个性化推荐。
使用机器学习模型预测菜品销量，辅助食堂科学备餐。
系统不仅展示数据，还能输出经营建议，具有决策支持功能。
注重隐私保护，使用匿名学生编号进行分析。
## 预期结果

系统完成后，预计可以实现以下结果：

清晰展示校园食堂整体经营情况。
找出最受欢迎和最不受欢迎的菜品。
发现学生在早餐、午餐、晚餐和夜宵的不同行为规律。
挖掘出高价值菜品组合。
将学生划分为不同消费群体。
为学生推荐个性化菜品。
预测未来菜品销量。
为食堂提供备餐和经营优化建议。
